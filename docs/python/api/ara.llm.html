ara.llm.context module¶

Conversation context manager with slice-based entity visibility.

Each entity (character) only sees messages from rounds in which they were present in the scene. This mimics real-world conversations where off-scene characters miss dialogue.

assistant_message(content: str, tool_calls: list[dict] | None = None, name: str = 'System', reasoning_content: str = '') → Self[source]¶

Append an assistant message.

DeepSeek thinking-mode compliance is handled automatically:

  • If tool_calls is non-empty, reasoning_content is preserved in the message dict because the API requires it on subsequent turns that involve tool calls.

  • If tool_calls is empty, reasoning_content is omitted because the API ignores it on non-tool turns.

If the previous message was from an assistant, an empty user padding message is inserted automatically to preserve valid API sequencing.

Parameters:

  • content – Assistant text.

  • tool_calls – List of completed tool-call descriptors.

  • name – Speaker name.

  • reasoning_content – Raw chain-of-thought text (may be empty).

ara.llm.tools module¶

Tool schema builder and lightweight hook registry.

class ara.llm.tools.ToolRegistry(_hooks: dict[str, ~typing.Callable[[...], str]] = <factory>)[source]¶

Bases: object

Simple name-to-callable mapping for tool execution.

The orchestrator and character agents register their tool implementations here. After the LLM returns tool calls, the registry dispatches each call to its registered Python function.
